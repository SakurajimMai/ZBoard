# 产品需求文档

## 产品定位

Zboard 是一个自研商业订阅节点中控系统，面向需要售卖网络节点订阅、管理用户套餐、控制节点配置、统计用户流量和维护节点运行时的运营方。

系统由以下能力组成：

- 用户系统
- 套餐系统
- 订单系统
- 支付系统
- 订阅系统
- 节点系统
- Agent 系统
- 配置生成系统
- 流量统计系统
- 后台管理系统

底层节点运行时可以使用 Xray-core 或 sing-box core，但它们只承担执行引擎职责。节点管理、配置生成、流量聚合、权限控制和商业逻辑全部由 Zboard 自研完成。

## 非目标

- 不依赖 3x-ui。
- 不兼容 3x-ui。
- 不做 3x-ui Adapter。
- 不读取 3x-ui 数据库。
- 不调用 3x-ui API。
- 不保留 3x-ui 迁移逻辑。
- MVP 不做复杂节点智能调度和灰度发布。

## 用户角色

| 角色 | 说明 | 核心需求 |
| --- | --- | --- |
| 游客 | 未登录访问者 | 查看套餐、注册账号 |
| 普通用户 | 已购买或待购买用户 | 购买套餐、查看订阅、查看流量、续费 |
| 管理员 | 运营后台用户 | 管理用户、套餐、订单、支付、节点、流量和系统配置 |
| Node Agent | 部署在 VPS 上的节点代理 | 注册、心跳、拉任务、应用配置、上报流量 |
| 支付平台 | 外部支付渠道 | 发送支付回调，完成订单状态变更 |

## MVP 必做

- 用户注册 / 登录。
- 套餐展示与购买。
- 订单创建与支付回调。
- 支付回调验签、金额校验和幂等处理。
- 后台添加套餐。
- 后台添加节点和节点组。
- Node Agent 注册。
- Node Agent 心跳。
- 订阅 token 生成和重置。
- 节点配置生成。
- Agent 拉取完整配置并应用。
- Agent 上报流量增量。
- 到期自动禁用。
- 超额自动禁用。
- 后台查看用户、订单、节点、任务和流量。

## MVP 暂不做

- 邀请返佣。
- 复杂优惠码。
- 企业套餐。
- 多支付渠道。
- 自动工单分派。
- 高级风控。
- 多语言。
- 移动 App。
- 复杂节点智能调度。
- Agent 远程升级。
- 配置灰度发布。

## 最小业务闭环

```text
用户付款
  ↓
系统开通套餐
  ↓
生成订阅 token
  ↓
生成节点配置
  ↓
Agent 同步配置
  ↓
用户订阅可用
  ↓
Agent 上报流量
  ↓
超额 / 到期自动停用
```

- 节点名称。
- 节点地区。
- 节点状态。
- Agent 版本。
- 运行时类型。
- 运行时状态。
- 最近心跳。
- 当前在线人数。
- 今日流量。
- 总流量。
- 配置版本。
- 最近同步时间。

- 禁用节点。
- 维护模式。

- 回滚配置版本。

## 开发阶段

### 阶段 0：产品和架构冻结

产出：

- PRD。
- 数据库设计。
- API 文档。
- Agent 协议文档。
- 订阅格式文档。
- 节点配置模型。
- 部署架构图。
- 安全基线。

### 阶段 1：商业主系统

开发顺序：

- AuthModule。
- UserModule。
- PlanModule。
- OrderModule。
- PaymentModule。
- SubscriptionModule。
- AdminModule。
- AuditLogModule。

目标：

- 用户可以注册。
- 用户可以购买套餐。
- 支付成功后可以开通套餐。
- 用户可以看到订阅入口。
- 后台可以管理用户、套餐和订单。

### 阶段 2：节点系统

开发内容：

- NodeModule。
- NodeGroupModule。
- NodeAgentModule。
- NodeTaskModule。
- RuntimeConfigModule。
- ConfigBuilder。
- Agent Gateway。

目标：

- 后台可以添加节点。
- 节点可以安装 Agent。
- Agent 可以注册。
- Agent 可以心跳。
- 系统可以生成配置任务。
- Agent 可以拉取并执行任务。

### 阶段 3：订阅生成服务

MVP 优先支持：

- Clash Meta。
- sing-box。
- 通用 Base64。

目标：

- 有效用户返回节点。
- 过期用户不返回节点。
- 超额用户不返回节点。
- 维护节点不返回。
- 不同客户端返回不同格式。

### 阶段 4：流量统计

开发内容：

- Agent 流量采集。
- 流量增量上报。
- 后端聚合。
- 用户流量快照。
- 超额检查任务。
- 禁用任务下发。

目标：

- 后台可看用户流量。
- 后台可看节点流量。
- 超额自动停用。
- 到期自动停用。

### 阶段 5：管理后台完善

优先页面：

- 数据总览。
- 用户管理。
- 套餐管理。
- 订单管理。
- 节点管理。
- Agent 状态。
- 节点任务。
- 流量统计。
- 操作日志。

### 阶段 6：安全、监控、稳定性

补齐：

- 管理员 2FA。
- API 限流。
- 支付回调验签。
- Agent 请求签名。

- Loki 日志。
- 数据库备份。
- 配置回滚。
- 异常告警。

## 上线验收标准

- 用户支付成功后能自动开通套餐。
- 用户能拿到有效订阅链接。
- Agent 能注册、心跳、拉取任务、应用配置并上报结果。
- 节点配置变更失败时可以回滚。
- 用户流量能按增量上报并聚合。
- 到期和超额用户会被停止返回订阅节点。
- 支付回调、Agent 请求和后台敏感操作都有审计记录。
- PostgreSQL、Redis 和管理端口不直接暴露公网。
