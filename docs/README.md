# Zboard 文档中心

Zboard 是一个纯自研商业订阅节点中控系统。它由控制面、节点 Agent、订阅生成、节点运行时管理、流量统计和后台管理组成，目标是直接管理节点运行时，而不是依赖现成面板。

## 文档状态

当前文档处于架构冻结前的第一版，作用是把 README 里的想法整理成后续可以直接开发的规格。当前第一阶段已先由 API Server 同源提供用户控制台 `/app` 和管理工作台 `/admin`，后续复杂交互再迁移到独立前端工程。正式编码前，需要优先确认：

- 后端技术栈最终选择。
- 支付渠道和回调协议。
- 第一版支持的节点协议和客户端格式。
- Agent 生产部署方式。
- Docker Compose 的服务拆分边界。

## 核心原则

- 不依赖 3x-ui、V2Board、Xboard 或其他现成面板。
- 不兼容 3x-ui，不读取 3x-ui 数据库，不调用 3x-ui API。
- 控制面保存商业数据、用户权限、节点配置和任务状态。
- 节点 Agent 主动连接控制面，中央系统不主动 SSH 到节点。
- Xray-core / sing-box 只作为运行时执行引擎，不作为管理面板。
- MVP 优先跑通付款、开通、配置同步、订阅可用、流量统计、到期/超额停用链路。
